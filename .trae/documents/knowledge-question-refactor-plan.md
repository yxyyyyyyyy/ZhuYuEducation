# 教材管理 + 题库结构重构计划

## 需求分析

### 问题 1：教材管理知识点操作冗余
- 顶部已选了年级+学科，知识点操作表单里还有「学科」「年级」输入框 → 冗余
- 知识点树层级过多（三级），用户希望简化为两级：
  - **第一级**：年级+教材（自动生成，不可编辑，即教材根节点）
  - **第二级**：知识点（用户创建）
- "基础知识"/"核心方法"这类标签不应作为树的第三级，而是作为**题目标签（tags）**

### 问题 2：题库表头与数据不匹配
- 当前表头：一级分类 / 二级分类 / 三级知识点
- 应改为：年级 / 学科 / 知识点 / 标签（tags）

---

## 步骤一：教材管理 — 简化知识点操作表单

### HTML 改动 (`admin.html`)
- 删除「层级」下拉框（不再需要，只有二级知识点）
- 删除「学科」输入框（从顶部年级+学科筛选自动继承）
- 删除「年级/学段」输入框（同上）
- 删除「三级关联题库知识点」下拉框（不再需要三级）
- 保留：父节点（即教材根节点）、名称、标签（新增，逗号分隔，如"基础知识,核心方法"）
- 表单标题改为「知识点管理」

### JS 改动 (`admin.js`)
- `getKnowledgeFormPayload()`：移除 level/subject/grade_level 字段，自动从当前教材继承年级+学科，level 固定为 2
- `createKnowledgeNode()` / `updateKnowledgeNode()`：携带 tags 字段
- `resetKnowledgeForm()`：清理标签输入框
- `startEditKnowledgeNode()`：回填标签

---

## 步骤二：知识点树简化为两级

### JS 改动 (`admin.js` → `renderKnowledgeTree`)
- 第一级（level=1）：显示为「年级 · 学科 · 教材名」，不可编辑/删除，显示"教材目录"标签
- 第二级（level=2）：显示知识点名称 + 标签（如有），可编辑/删除
- node-meta 信息简化：第一级显示年级+学科+教材名，第二级显示标签+题目数

---

## 步骤三：题库表头重构

### HTML 改动 (`admin.html`)
- 表头从「一级分类 / 二级分类 / 三级知识点」改为「年级 / 学科 / 知识点 / 标签」

### JS 改动 (`admin.js` → `renderQuestionTable`)
- 表格列：年级(item.grade_level) / 学科(item.subject) / 知识点(item.topic_name) / 标签(item.tags)
- tags 显示为小标签（pill 形状）

---

## 涉及文件

| 文件 | 改动类型 |
|------|---------|
| `app/templates/admin.html` | HTML 结构（知识点表单、题库表头） |
| `app/static/admin.js` | JS 逻辑（表单提交、树渲染、表格渲染） |

不涉及后端改动。tags 字段后端已支持（Question 模型有 `tags: List[str]`）。
