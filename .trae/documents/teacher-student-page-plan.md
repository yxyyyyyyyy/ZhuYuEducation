# 教师端修复计划：模拟数据修正 + 学生列表独立页面

## 问题分析

### 问题1：模拟数据不合理
- **当前**：8个学生全部分布在初一/初二/初三，但班级只有"八年级1班"
- **当前**：所有学生的 `target_subject` 都是"数学"
- **应该**：学生都应该是初二的（匹配班级），且学科应多样化（语文、英语、数学、物理等）

### 问题2：学生列表应独立成页面
- **当前**：学生列表嵌在"数据总览"页面的左半边，空间太小
- **应该**：学生列表作为独立页面，有充足空间展示年级-班级分组和学生详情

---

## 实施步骤

### 步骤1：修正种子数据（container.py）

修改 `demo_student_names` 列表：
- 所有学生年级统一为"初二"（匹配八年级1班/2班）
- 学科分布多样化：数学、语文、英语、物理等
- 增加第二个班级"八年级2班"
- 学生数量增加到 10-12 个，分布更合理

### 步骤2：学生列表独立页面

**修改 teacher.html：**
1. 在侧边栏导航中添加"学生管理"页面入口（概览组下方）
2. 新增 `<section class="page" data-page="students">` 页面：
   - 页面标题 + 刷新按钮
   - 年级-班级分组卡片列表（复用之前的 class-group-card 样式）
   - 右侧抽屉查看班级学生详情（复用已有 Drawer）
3. 从 dashboard 页面移除学生列表区域，只保留 KPI 卡片 + 练习分析

**修改 teacher.js：**
1. 移除 `renderStudentList` 从 `renderTeacherDashboard` 中的调用
2. 新增 `renderStudentPage()` 函数：独立页面的学生列表渲染逻辑
3. 在 `bootstrap` 和 `login` 流程中加载学生数据到独立页面
4. 在 `navigateTo` 中处理 students 页面的数据加载

### 步骤3：删除旧数据库重建

由于种子数据的去重逻辑，旧学生记录不会被更新。需要删除 `data/` 下的 SQLite 数据库文件，让系统重新初始化种子数据。
